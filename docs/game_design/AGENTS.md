# Game Design Documentation - AI Agent Guidelines

## Purpose

Documents describing gameplay mechanics and user experience.

## Audience

- Game designers
- Players wanting to understand mechanics
- Developers implementing features

## Document Purposes

### gameplay_overview.md

Core game experience:

- What players do
- Game modes
- Progression (if any)

### world_generation.md

Procedural content:

- Terrain algorithms
- Biome distribution
- Feature placement

### block_system.md

Building system:

- Shape types
- Placement rules
- Properties

### controls_and_navigation.md

Input handling:

- Key mappings
- Mouse controls
- Navigation system

### advanced_building_techniques.md

Building in tetrahedral space:

- Slope, angle, and structural-integrity principles for non-cubic voxels
- Named building patterns (helix, geodesic dome, smooth tunnels)
- Techniques for stable, aesthetically pleasing structures

## Guidelines

### Updating for Gameplay Changes

1. Document intended behavior
2. Note differences from Minecraft (familiar reference)
3. Include screenshots where helpful
4. Keep controls section accurate

### Balancing Considerations

When documenting gameplay values:

- Note current values
- Explain rationale
- Suggest tuning ranges

### Player Perspective

Write as if explaining to new players:

- Clear, accessible language
- Define tetrahedral/quadray concepts
- Provide context for unique mechanics
